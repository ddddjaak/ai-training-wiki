# IDE 集成

## 概述

如果你习惯在 IDE 里工作而不是纯终端，Claude Code 提供了 VS Code 和 JetBrains 两个主流 IDE 的插件。装在 IDE 里和终端里功能完全一样——区别只是你把对话放到了侧边栏。

> **选择建议**：日常开发推荐用 IDE 插件（不用切窗口），调试和跑复杂工作流推荐回终端（输出更完整、命令响应更快）。

---

## VS Code

### 安装

1. 打开 VS Code，点击左侧活动栏的「扩展」图标（或按 `Ctrl+Shift+X`）
2. 搜索 **Claude Code**
3. 找到 Anthropic 官方发布的扩展，点击「安装」

> 📷 **截图**：VS Code 扩展商店搜索 Claude Code（待补充截图）

### 使用

1. 安装完成后，按 `Ctrl+Shift+P` 打开命令面板
2. 输入 `Claude Code: Open` 并回车
3. Claude Code 面板出现在右侧/底部——和终端里的 `claude` 命令完全相同

> 📷 **截图**：VS Code 中 Claude Code 面板（待补充截图）

### 常用快捷键

| 操作 | 快捷键 |
|------|--------|
| 打开 Claude Code 面板 | `Ctrl+Shift+P` → `Claude Code: Open` |
| 在对话中引用当前文件 | 在面板中输入 `@` 然后选文件 |

## ⚠️ 安装后 Claude Code 不出现？排查清单

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| VS Code 搜不到 Claude Code 扩展 | 扩展名称输入不对 | 确认搜索 "Claude Code"，发布者是 Anthropic |
| 安装后 VS Code 没有 Claude Code 面板 | 没有手动打开 | 按 `Ctrl+Shift+P` → 输入 `Claude Code: Open` |
| JetBrains 安装后找不到 Claude Code | 忘记重启 IDE | 重启 IDE。如果还找不到：**View → Tool Windows → Claude Code** |
| 面板打开了但输入命令无响应 | Claude Code CLI 未安装或 CC Switch 未启用 | 回到终端运行 `claude` 确认能正常启动。如果终端可以但 IDE 不行，重启 IDE |
| IDE 插件提示 "Claude Code CLI not found" | 插件找不到 `claude` 命令的安装路径 | 确认 `claude --version` 在终端能输出版本号。如果终端可以：在 IDE 设置中找到 Claude Code 配置，手动指定 Claude Code CLI 路径 |

## IDE vs 终端：什么时候用哪个

| 场景 | 推荐 | 原因 |
|------|------|------|
| 日常开发（边写代码边问 AI） | IDE 插件 | 不用切窗口，`@` 引用当前文件方便 |
| 调试（长对话 + 跑编译测试） | 终端 | 输出完整，命令响应直接 |
| Code Review | IDE 插件 | 在代码旁边看审查结果 |
| 复杂工作流（多步操作 + 多个工具调用） | 终端 | 交互更快，工具栏输出更直观 |
| 后面讲的 skill 触发 | 都可以 | skill 在终端和 IDE 里触发效果一样 |
